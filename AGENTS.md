# Codex 作業ルール

このファイルは、このプロジェクトで Codex が作業するときの基本ルールをまとめたものです。

## 基本方針

- 既存コードや設定を大きく壊さないように、必要最小限の変更を行います。
- 変更する前に、関連するファイルを確認してから作業します。
- 変更後は、可能であればテストや動作確認を行います。
- 実行したコマンドや変更した理由は、簡潔に説明します。
- README やコメントは、初心者にも分かる言葉で書きます。

## Windows 環境での注意

- このプロジェクトは Windows 環境で作業される前提を考慮します。
- コマンド例を書くときは、PowerShell で実行しやすい形を優先します。
- パス区切りや環境変数の書き方が OS によって違う場合は、Windows で分かりやすい説明を加えます。

## 不明点がある場合

- 仕様や目的が不明な場合は、勝手に大きな仕様変更をしません。
- 判断に迷う変更は、作業前に確認します。
- 既存の重要な説明や設定は、理由なく削除しません。
