# MVP 開発チケット

## 実装方針

- チケットは 1～3 日程度で完了できる粒度を目安にする。
- 各機能チケットには、API、認可、画面、入力検証、自動テストを含める。
- 実装順は依存関係を優先し、PDF や可視化は元データの登録フロー完成後に着手する。

## Epic 一覧

| Epic    | 名称                 | 完了の定義                                           |
| ------- | -------------------- | ---------------------------------------------------- |
| EPIC-01 | 基盤・認証           | 安全にログインし、ユーザー単位でデータを分離できる。 |
| EPIC-02 | キャリア・スキル     | キャリア目標と必要スキルを登録し、差分を確認できる。 |
| EPIC-03 | ロードマップ・目標   | ロードマップから測定可能な目標を作成できる。         |
| EPIC-04 | 日次実績・証跡       | 日々の行動・成果・証跡を目標へ記録できる。           |
| EPIC-05 | 進捗・ダッシュボード | 達成率、遅延、活動状況を確認できる。                 |
| EPIC-06 | 評価資料             | 評価期間で集約し、Web と PDF で出力できる。          |
| EPIC-07 | 品質・リリース       | 非機能要件、E2E、運用文書が整う。                    |

## 推奨リリース分割

| マイルストーン | 対象         | 利用者が得られる価値                           |
| -------------- | ------------ | ---------------------------------------------- |
| M1 基盤        | MVP-001～005 | 安全に利用を開始できる。                       |
| M2 計画        | MVP-010～024 | キャリアから具体的目標まで計画できる。         |
| M3 記録        | MVP-030～043 | 日々の記録から進捗と遅延を把握できる。         |
| M4 評価        | MVP-050～053 | 評価資料を Web と PDF で作成できる。           |
| M5 MVP 完了    | MVP-060～063 | 復元性・安全性・運用性を含めて公開判断できる。 |

## チケット共通の完了条件

- 受入条件を満たす自動テストがある。
- ユーザー所有データの API は認可テストを持つ。
- 入力エラーと空状態を画面で確認できる。
- PC とスマートフォン幅で操作できる。
- 仕様変更がある場合は本ディレクトリの該当文書を更新する。
- lint、format、build、test が成功する。
