本ディレクトリは、要件定義を MVP として実装できる粒度へ整理する。

## 前提

- 初期版は個人利用だが、データ分離を保証するためユーザー認証を実装する。
- PC とスマートフォンのレスポンシブ Web アプリとする。
- 添付ファイルは MVP 対象外とし、証跡は URL 登録に限定する。
- 週次・月次の振り返り専用機能は MVP 後とし、評価資料の自由記述欄で代替する。
- 一覧の削除は論理削除またはアーカイブを基本とし、誤削除から復元できる設計を採る。

## 識別子

- `SCR-xxx`: 画面
- `EPIC-xx`: 開発 Epic
- `MVP-xxx`: 開発チケット
- `BR-xx`: 業務ルール
- `NFR-xx`: 非機能要件
