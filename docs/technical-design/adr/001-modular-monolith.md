# ADR-001: モジュラーモノリスと4層構成を採用する

- 状態: Proposed
- 決定日: 2026-07-11

## コンテキスト

MVP は個人利用であり、少人数で TDD と DDD を用いて開発する。将来 EKS に配置する可能性はあるが、現時点でサービスごとの独立スケールや組織分割は必要ない。

## 決定

単一 API のモジュラーモノリスとし、プレゼンテーション、アプリケーション、ドメイン、インフラストラクチャへ分ける。業務領域の境界もコード上で分ける。

## 結果

- 分散トランザクションやサービス間通信なしで MVP を開発できる。
- ドメイン層をフレームワークから独立させ、高速な単体テストを書ける。
- レイヤーが形だけにならないよう、import 制約とレビューが必要になる。
- 将来のサービス分割は可能だが、必要性が生じるまで実施しない。
